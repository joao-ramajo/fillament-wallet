# Regras De Testes

## Regras base
- Toda alteração de fluxo relevante deve considerar cobertura por teste de feature ou unitário.
- Novas rotas devem preferencialmente ter testes cobrindo sucesso, validação e falhas principais.
- Correções de bug devem incluir um teste que reproduza o problema corrigido.

## Regras pendentes de definição
- Critério para decidir entre teste de `Feature` e `Unit`.
- Política mínima de cobertura para módulos críticos.
- Uso de factories, seeders e dados fixos em testes.
- Estratégia para testes de integração com autenticação, fila, mail e exportação.
